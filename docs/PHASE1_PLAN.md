# 머지 머니 타이쿤 - Phase 1 라이브 운영 기획서 (Live Ops Phase 1)

이 문서는 기존의 "출시 준비용 확장 기획"이 아니라,
**이미 Google Play 스토어에 등록된 라이브 게임을 안정적으로 업데이트하기 위한 1차 운영 계획서**입니다.

## 1. 현재 전제
- 게임은 이미 Google Play 콘솔에서 비공개 테스트를 마쳤습니다.
- 프로덕션 심사 및 스토어 등록까지 완료된 상태입니다.
- 따라서 앞으로의 개발은 "출시 준비"보다 **라이브 운영 / 안정화 / 리텐션 강화 / 향후 수익화 준비**가 중심입니다.

---

## 2. 이번 Phase 1의 목표

### 목표 A — 라이브 안정성 확보
- 저장/로드 안정성 재검토
- 버전업 시 세이브 호환성 리스크 점검
- 주요 UI 흐름의 오류 가능성 축소

### 목표 B — 리텐션 구조 1차 도입
- 유저가 다시 접속할 이유를 만든다.
- 단순 콘텐츠 추가보다 **반복 접속 동기**를 우선한다.

### 목표 C — 향후 수익화 준비
- 당장 과한 수익화 도입보다는,
  **광고/결제 구조를 붙여도 게임성이 무너지지 않는 기반**을 먼저 마련한다.

---

## 3. 1차 후보 기능

### 3.1 일일 보상 / 복귀 보상
- 하루 1회 접속 보상
- 일정 시간 이상 미접속 후 복귀 시 보너스
- 초반에는 단순한 구조로 시작하고, 추후 streak로 확장 가능

### 3.2 오프라인 보상 체감 개선
- 기존 오프라인 보상이 있다면 연출/가독성/수치 체감 보강
- 없다면 최소형 오프라인 보상 구조를 우선 검토

### 3.3 미션형 목표 시스템
- 일일 미션 / 누적 목표 / 업적 재정렬
- 유저가 현재 무엇을 해야 할지 명확히 보이도록 구성

### 3.4 중장기 메타 준비
- 환생(Prestige), 투자 시스템, 도감은 여전히 유효한 방향이지만,
  Phase 1에서는 **바로 대형 도입**보다 설계 정리와 기반 준비를 우선한다.

---

## 4. 기술적 우선순위

### 4.1 상태/세이브 구조 점검
- 저장 포맷 버전 관리 여부 확인
- 신규 필드 추가 시 마이그레이션 전략 정리
- 세이브 초기화 방지 로직 점검

### 4.2 UI/UX 미세 개선
- 보상 수령 동선 단축
- 버튼/모달 중복 진입 방지
- 초반 플레이 피로도 완화

### 4.3 운영 문서 정리
- 로드맵 최신화
- 개인정보/문의 경로 문서 최신화
- 다음 업데이트 단위별 TODO 정리

---

## 5. 추천 작업 순서

1. **문서 및 운영 상태 정리**
2. **라이브 안정성 점검 및 자잘한 UX 개선**
3. **리텐션 기능 1개 우선 구현**
   - 추천 우선순위: 일일 보상 > 복귀 보상 > 미션 시스템
4. **수익화 후보 구조 설계 문서화**
5. **중장기 메타(환생/투자/도감) 설계 갱신**

---
