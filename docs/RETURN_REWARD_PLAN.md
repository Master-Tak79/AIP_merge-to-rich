# 복귀 보상 시스템 설계안 (Return / Comeback Reward)

작성일: 2026-03-08  
상태: 설계 완료 (구현 전)

## 1) 제품 의도
- 최근 업데이트로 도입된 일일 보상과 별개로, 일정 기간 미접속 유저의 재접속 동기를 강화합니다.
- 복귀 직후 "손해를 만회했다"는 체감을 주되, 기존 활성 유저 대비 과도한 우위를 주지 않습니다.
- 라이브 운영 관점에서 저위험/저복잡도 규칙부터 적용하고, 데이터 확인 후 확장합니다.

## 2) 대상 및 자격 규칙
- 기준 시간대: KST(Asia/Seoul)
- `복귀 대상`: 마지막 접속(또는 마지막 보상 수령) 기준 **48시간 이상** 미접속
- `수령 빈도`: 복귀 판정 1회당 1회 수령
- `쿨다운`: 복귀 보상 수령 후 다시 48시간 이상 미접속해야 재대상

## 3) 보상 구조 (초기안)
- 기본 보상: `현재 spawnLevel 기준 코인 가치 x N`
- 권장 초기값:
  - 48h~72h 미접속: `x 30`
  - 72h~168h 미접속: `x 60`
  - 168h+ 미접속: `x 120`
- 상한: 초반 밸런스 붕괴 방지를 위해 1회 보상 최대값 캡 적용
  - 예: `max(기준 코인 가치 x 300, 5,000,000)` 중 작은 값 사용

## 4) 악용 방지
- 디바이스 시간 변경 완화:
  - 저장값에 `lastSeenAt`와 `lastSeenDayKey(KST)`를 함께 저장
  - 현재 시간이 과거로 역행하면 보상 지급 보류 + 경고 로그
- 반복 재설치/저장삭제 악용:
  - 로컬 기반 한계는 인정하되, 초기 버전은 과도한 방어보다 오탐 최소화 우선
- 중복 수령 방지:
  - `returnRewardLastClaimAt` 기록 후 동일 복귀 창에서 재수령 금지

## 5) UX 플로우
- 앱 진입 시 eligibility 체크
- 대상이면 `복귀 환영 모달` 1회 노출
  - 미접속 시간, 수령 금액, 수령 버튼 표시
- 수령 후:
  - 자산 즉시 반영
  - 짧은 토스트 표시
  - 같은 세션에서 재노출 금지

## 6) 저장 모델 제안
- `returnRewardLastEligibleAt: number | null`
- `returnRewardLastClaimAt: number | null`
- `returnRewardTotalClaimed: number`
- `lastSeenAt: number`
- `lastSeenDayKey: string | null`

모든 필드는 기존 `zustand/persist` `partialize` 경로에 포함합니다.

## 7) 구현 메모
- 일일 보상 유틸(`KST day key`)과 동일한 시간 계산 유틸을 재사용
- 구현 순서:
  1. store 상태/액션 추가
  2. 앱 부팅 시 eligibility 계산
  3. 모달 UI 연결
  4. 저장/복원 마이그레이션 fallback 추가
- 위험도:
  - 로직 자체는 저위험
  - 밸런스 수치(N, 캡)는 라이브 데이터 보고 보정 필요

## 8) 출시 체크리스트 (복귀 보상)
- build 통과
- 저장/복원 후 eligibility 유지 확인
- 같은 창에서 중복 수령 불가 확인
- 시간 역행 시 과지급 방지 확인
- 한국어 문구/수치 표기 검수
