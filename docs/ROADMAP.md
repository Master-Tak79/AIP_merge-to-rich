# 머지 머니 타이쿤 라이브 운영 로드맵 (Post-Launch Roadmap)

본 문서는 **Google Play 비공개 테스트 종료 → 프로덕션 통과 → 심사 완료 → 스토어 등록 완료** 이후의 운영 방향을 정리한 라이브 서비스 기준 로드맵입니다.

이제 이 프로젝트의 목표는 단순 출시가 아니라,
**안정적으로 업데이트를 이어가며 리텐션을 높이고, 이후 수익화 기반을 무리 없이 붙일 수 있는 구조를 만드는 것**입니다.

---

## 핵심 운영 원칙

### 1. 라이브 게임 우선
- 신규 기능보다 먼저 **저장 안정성, 밸런스, 퍼포먼스, 체감 UX**를 우선 점검합니다.
- 이미 스토어에 올라간 게임이므로, "재미 있는 기능"보다 "망가지지 않는 업데이트"가 더 중요합니다.

### 2. 리텐션 먼저, 수익화는 그 다음
- 유저가 **다시 접속할 이유**가 먼저 만들어져야 수익화도 건강하게 붙습니다.
- 단기적으로는 광고/결제보다 **복귀 동기, 반복 플레이 가치, 성장 체감**을 먼저 강화합니다.

### 3. 소규모-고빈도 업데이트 전략
- 한 번에 큰 개편보다, **짧은 주기의 작은 개선**이 라이브 게임 운영에 더 유리합니다.
- 작은 기능 1~2개 + 안정화 + 빌드 검증 + 체인지로그 정리 방식으로 운영합니다.

### 4. 개발 워크플로우는 로컬 우선
- 개발은 우선 **로컬에서 충분히 진행**합니다.
- GitHub push는 필요 시점(안정 마일스톤, 백업 시점, 배포 준비 시점)에 수행합니다.
- 단, 로컬에서도 **작은 단위 커밋은 자주** 남겨서 회귀 추적 가능성을 확보합니다.

---

## Phase A — 라이브 안정화 & 운영 기반 정비 (지금 ~ 단기)

### 목표
- 현재 라이브 버전을 무리 없이 유지하면서, 앞으로의 업데이트 속도를 올릴 수 있는 기반을 정리합니다.

### 주요 과제
1. **저장/로드 안정성 재점검**
   - 로컬 스토리지 저장 실패/초기화 케이스 점검
   - 버전업 시 세이브 호환성 점검

2. **핵심 UX 미세 개선**
   - 합성 피드백, 버튼 반응성, 보상 체감, 정보 밀도 정리
   - 튜토리얼/초반 진입 흐름 재점검

3. **밸런스 재조정**
   - 초반 성장 속도
   - 중반 정체 구간
   - 후반 목표 도달 시간
   - 방치 보상 체감

4. **운영 문서 정리**
   - 로드맵, 기획 문서, 변경 이력, 배포 체크리스트를 라이브 운영 기준으로 갱신

### 완료 기준
- 로컬 빌드/실행 안정
- 문서 기준 최신 운영 상태 반영
- 다음 업데이트용 체크리스트 정리 완료

---

## Phase B — 리텐션 강화 (단기 ~ 중기)

### 목표
- "한 번 해보고 끝"이 아니라 **다시 켜게 만드는 구조**를 강화합니다.

### 후보 기능
1. **출석/복귀 보상 시스템**
   - 일일 보상
   - 복귀 보너스
   - 연속 접속 보상

2. **일일/주간 미션**
   - 합성 횟수, 특정 코인 도달, 누적 수익 등
   - 플레이 목적을 짧고 명확하게 제공

3. **오프라인 보상 개선**
   - 단순 수익 지급 외에도 복귀 연출/보너스 추가 검토

4. **경성 메타 성장 도입**
   - 기존 플레이를 무효화하지 않는 선에서 성장 축 추가
   - 예: 영구 보너스, 해금형 도감, 보너스 목표

### 성공 기준
- 재접속 동기 증가
- 플레이 타임 및 반복 플레이 가치 상승

---

## Phase C — 수익화 기반 도입 (중기)

### 목표
- 유저 경험을 해치지 않는 선에서 **초기 수익화 구조**를 설계합니다.

### 원칙
- Pay-to-win 인상 최소화
- 광고 강제 노출 지양
- "편의/가속/보너스" 중심 구조 우선 검토

### 후보 수익화 요소
1. **보상형 광고 (Rewarded Ads)**
   - 오프라인 보상 2배
   - 일시 버프
   - 실수 복구/추가 보상

2. **광고 제거 패키지**
   - 장기적으로 가장 거부감이 적은 상품 후보

3. **스타터 팩 / 소액 상품**
   - 초반 이탈 방지 + 과한 밸런스 붕괴 없는 범위

4. **이벤트성 한정 상품**
   - 추후 라이브 운영이 안정화된 뒤 도입 검토

### 선행 필요 조건
- 리텐션 구조 최소 1개 이상 정착
- 핵심 저장/진행 흐름 안정화
- 개인정보처리방침/스토어 문구/정책 재검토

---

## Phase D — 메타 확장 & 장기 스케일업 (중장기)

### 목표
- 게임의 수명을 늘리는 중장기 성장축을 추가합니다.

### 주요 후보
1. **환생(Prestige)**
   - 반복 플레이의 장기 목표 제공

2. **투자/경영 탭**
   - 오프라인 수익 축 확장

3. **도감/수집형 콘텐츠**
   - 해금 욕구 강화

4. **이벤트/테마 시즌 운영**
   - 주기적 복귀 동기 확보

5. **장르 혼합형 확장 모드**
   - 머지 + 방치 RPG
   - 머지 + 디펜스
   - 머지 + 꾸미기

## 당장 추천하는 우선순위

### 우선순위 1
- 라이브 기준 문서 정리
- 현재 버전 안정성/밸런스 점검
- 다음 업데이트 단위 정의

### 우선순위 2
- 리텐션 기능 1개 우선 구현
  - 추천: **일일 보상** 또는 **복귀 보상**

### 우선순위 3
- 수익화 후보를 문서화만 먼저 해두고, 실제 적용은 리텐션 보강 후 진행

---

## 권장 작업 방식

- 개발/실험: **로컬 우선**
- 커밋: **작은 단위로 자주**
- 원격 push: **요청 시 또는 안정 마일스톤 시점**
- 배포 전: 빌드 + 기본 플레이 + 저장/로드 + 주요 UI 스모크 테스트 필수
